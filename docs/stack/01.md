# The Celebrity Problem

Given a square matrix `mat[][]` of size $n \times n$, where `mat[i][j] = 1` means person $i$ knows person $j$, find the celebrity person.

A celebrity person is one who:
- known by everyone else
- does not know anyone (except themselves)

## Example

$$
\text{input} = \begin{bmatrix}
1
\end{bmatrix}, \ \text{output} = 0
$$ 


$$
\text{input} = \begin{bmatrix}
1 & 1 \\
1 & 1
\end{bmatrix}, \ \text{output} = -1
$$ 


$$
\text{input} = \begin{bmatrix}
1 & 1 & 0 \\
0 & 1 & 0 \\
0 & 1 & 1
\end{bmatrix}, \ \text{output} = 1
$$

## Solution

??? "First approach"

    $O(n^2)$ time and $O(n)$ space.

    1. Maintain two lists `indegree` and `outdegree`. 
    2. Iterate over all pairs of nodes $(i, j)$ and if $m_{ij} = 1$ then increment $\text{outdegree}_i$ and $\text{indegree}_j$.
    3. Any node with $\text{outdegree} = 1$ and $\text{indegree} = n$ is our celebrity.

    ```kotlin
    fun findCelebrity(mat: Array<IntArray>): Int {  
      val indegree = IntArray(mat.size)  
      val outdegree = IntArray(mat.size)  
      
      for (i in 0 until mat.size) {  
        for (j in 0 until mat.size) {  
          if (mat[i][j] != 0) {  
            outdegree[i]++  
            indegree[j]++  
          }  
        }  
      }  
      
      for (i in 0 until mat.size) {  
        if (indegree[i] == mat.size && outdegree[i] == 1) return i  
      }  
      
      return -1  
    }
    ```

??? "Second approach"

    $O(n)$ time and $O(n)$ space.

    1. create a stack $s$ and add all nodes to it.
    2. while $|s| \ge 2$:
        1. pop two elements $i$ and $j$.
        2. if $i$ knows $j$ then $i$ cannot be the celebrity.
        3. otherwise, $j$ cannot be the celebrity.
    3. the last remaining element on $s$ is our candidate celebrity.
    4. verify that $n$ nodes know our candidate and our candidate knows only $1$ node (itself).

    ```kotlin
    fun findCelebrity(mat: Array<IntArray>): Int {  
      val stack = ArrayDeque<Int>(mat.size)  
      for (i in 0 until mat.size) stack.add(i)  
      
      while (stack.size 1) {  
        val a = stack.removeLast()  
        val b = stack.removeLast()  
        if (mat[a][b] == 1) stack.add(b)  
        else stack.add(a)  
      }  
      
      val celebrity = stack.removeLast()  
      var whoKnowsCelebrity = 0  
      var whomCelebrityKnows = 0  
      for (i in 0 until mat.size) {  
        if (mat[celebrity][i] == 1) whomCelebrityKnows++  
        if (mat[i][celebrity] == 1) whoKnowsCelebrity++  
      }  
      
      return if (whoKnowsCelebrity == mat.size && whomCelebrityKnows == 1) celebrity else -1  
    }
    ```

??? "Third approach"

    $O(n)$ time and $O(1)$ space.

    1. we start two pointers $s = 0$ and $e = n-1$.
    2. while $s < e$:
        1. if $s$ knows $e$ then $s$ can't be the celebrity.
        2. otherwise, $e$ can't be the celebrity.
    3. the last remaining node ($s = e$) is our candidate celebrity.
    4. verify that n nodes know our candidate and our candidate knows only 1 node (itself).

    ```kotlin
    fun findCelebrity(mat: Array<IntArray>): Int {  
      var (i, j) = 0 to mat.lastIndex  
      while (i < j) {  
        if (mat[i][j] == 1) i++;  
        else j--  
      }  
      
      val celebrity = i  
      var whoKnowsCelebrity = 0  
      var whomCelebrityKnows = 0  
      for (i in 0 until mat.size) {  
        if (mat[celebrity][i] == 1) whomCelebrityKnows++  
        if (mat[i][celebrity] == 1) whoKnowsCelebrity++  
      }  
      
      return if (whoKnowsCelebrity == mat.size && whomCelebrityKnows == 1) celebrity else -1  
    }
    ```

    Let's clarify one thing, this is not your typical *two pointer* approach. As a matter of fact, we could've shuffled our nodes in any order we wanted, and this would've worked without a hitch.
    
    The pointers aren't looking for a "range" or a "sum". They are performing a Tournament-style Elimination. The "order" of the people doesn't matter; the pointers simply serve as a way to keep track of who is still "in the running."
