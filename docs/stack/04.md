# Infix to Postfix Expression

Given a string $s$ representing an infix expression, convert it to a postfix expression.

## Example

1. `a + b = a b +`
	- Simplest example.
2. `a + b * c = abc*+`
	- multiplication should happen before addition.
3. `a * (b + c) = abc+*`
    - parenthesis changed the priority of operation.
4. `a * b * c = ab*c*`
	- most operators `+, -, /, *` are left associative.
	- `a + b + c = (a + b) + c`.
5. `a ^ b ^ c = abc^^`
	- power operator `^` is right associative. 
	- `a ^ b ^ c = a ^ (b ^ c)`

## Solution

??? "Expand"

    We use _Shunting-Yard algorithm_ for converting an infix expression to postfix expression. The algorithm is broken in three parts:
    
    1. General handling of operators and operands.
    2. Exception for parenthesis.
    3. Exception for right associative operators.

    ### General handling of operators and operands
    1. If an _operand_ is encountered: simply pass it to the output.
    2. if an _operator_ is encountered:
        1. pop operators from the stack and pass them the output as long as $\text{operator on stack}\ge\text{incoming operator}$.
        2. push the incoming operator on stack.
    3. once all of the expression is processed, pop the whole stack to the output.

    ### Exception for parenthesis
    1. `(` is always pushed to the stack immediately. It acts as a "floor", i.e., it cannot be popped by any operator except `)`.
    2. `)` when encountered, pops the whole stack until a corresponding `(` is found.

    ### Exception for right associative operators
    This is necessary because we want to treat `2 ^ 3 ^ 4` as $2^{(3^4)} = 2^{81}$ and not ${(2^3)}^4 = 8^4$.

    - for normal left-associative operator: we pop stack if $I \ge S$.
    - for right-associative operator: we pop stack if $I < S$.

    where $I$ is the precedence of the incoming operator and $S$ is the precedence of the operator at the top of the stack. 

    === "Core"

        ```kotlin
        fun infixToPostfix(expression: String): String {  
          val stack = ArrayDeque<Char>()  
          val infix = StringBuilder()  
          
          for (token in expression) {  
            when {  
              token.isOperand -infix.append(token)  
              token == '(' -stack.add(token)  
              token == ')' -{  
                while (stack.isNotEmpty() && stack.last() != '(')  
                  infix.append(stack.removeLast())  
                stack.removeLast()  
              }  
              else -{  
                while (stack.shouldPop(token)) {  
                  infix.append(stack.removeLast())  
                }  
                stack.add(token)  
              }  
            }  
          }  
          
          while (stack.isNotEmpty())  
            infix.append(stack.removeLast())  
          
          return infix.toString()  
        }
        ```

    === "Checking when to pop"

        ```kotlin
        private fun ArrayDeque<Char>.shouldPop(incomingOperator: Char): Boolean {  
          val isRightAssociative = incomingOperator.isRightAssociative  
          return isNotEmpty() && if (isRightAssociative) {  
            incomingOperator.precedence < last().precedence  
          }  
          else {  
            incomingOperator.precedence <= last().precedence  
          }  
        }  
        ```

    === "Rest of utilities"

        ```kotlin
        private val Char.isRightAssociative: Boolean  
          get() = this == '^'  
          
        private val Char.isOperand: Boolean  
          get() = this !in listOf('+', '-', '/', '*', '^')  
          
        private val Char.precedence: Int  
          get() = when (this) {  
            '+', '-' -1  
            '*', '/' -2  
            '^' -3  
            else -0  
          }
        ```

## Unit tests

??? "Expand"

    ```kotlin
    import org.example.org.example.infixToPostfix  
    import org.junit.jupiter.api.Assertions.assertEquals  
    import org.junit.jupiter.api.Test  
      
    class InfixToPostfixTest {  
      
      // Helper to keep tests clean if your implementation doesn't handle spaces  
      private fun convert(input: String): String {  
        return infixToPostfix(input.replace(" ", ""))  
      }  
      
      @Test  
      fun `test simple addition and multiplication precedence`() {  
        // Multiplications should come before addition in postfix  
        assertEquals("abc*+", convert("a + b * c"))  
      }  
      
      @Test  
      fun `test parentheses overriding precedence`() {  
        // Parentheses force addition to happen first  
        assertEquals("ab+c*", convert("(a + b) * c"))  
      }  
      
      @Test  
      fun `test power operator associativity and precedence`() {  
        // Power (^) has higher precedence than multiplication  
        assertEquals("abcd^*+", convert("a + b * c ^ d"))  
      }  
      
      @Test  
      fun `test nested parentheses`() {  
        // Deeply nested brackets should be cleared in correct order  
        assertEquals("ab+c*d+", convert("((a + b) * c) + d"))  
      }  
      
      @Test  
      fun `test left-to-right associativity for same precedence`() {  
        // For + and -, the leftmost operator is processed first  
        assertEquals("ab-c+", convert("a - b + c"))  
      }  
      
      @Test  
      fun `test complex expression with various operators`() {  
        // A standard complex case found in most DSA textbooks  
        val infix = "a+b*(c^d-e)^(f+g*h)-i"  
        val expected = "abcd^e-fgh*+^*+i-"  
        assertEquals(expected, convert(infix))  
      }  
      
      @Test  
      fun `test single operand expression`() {  
        // A single character should return itself  
        assertEquals("x", convert("x"))  
      }  
      
      @Test  
      fun `test expression with division`() {  
        assertEquals("ab+cd/*", convert("(a + b) * (c / d)"))  
      }  
    }
    ```
