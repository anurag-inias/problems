# Implement Queue using Stack

Implement a queue using stack as the primary data structure. The queue should support the following operations.

## Placeholder

```kotlin
class Queue {

  // Insert element v at the end of the queue.
  fun enqueue(v: Int)
  
  // Remove the element from the front of the queue.
  fun dequeue(): Int?
  
  // Return the element at the front of the queue.
  fun front(): Int?
  
  // Number of elements in the queue.
  fun size(): Int

}
```

## Solution

??? "First approach"

    $O(n)$ enqueue and $O(1)$ dequeue.
    1. Maintain two stacks $s_1$ and $s_2$.
    2. on $\text{enqueue}(v)$, do following:
        1. $\text{pop}$ all elements from $s_1$ to $s_2$.
        2. $\text{push}$ new element $v$ to $s_1$.
        3. $\text{pop}$ back all elements from $s_2$ to $s_1$.
    
    For example, let's say we have already enqueued $q = [1, 2]$ elements. Internally, we will have:

    $$ 
    \begin{align}
    s_1 &= [2, 1] \\
    s_2 &= []
    \end{align}
    $$

    now when pushing a new element $3$, we first move all elements to $s_2$

    $$
    \begin{align}
    s_1 &= [] \\
    s_2 &= [1, 2]
    \end{align}
    $$

    Then we add $3$ to $s_1$ and put back the remaining elements.

    $$
    \begin{align} 
    s_1 &= [3, 2, 1] \\ 
    s_2 &= [] 
    \end{align}
    $$

    ```kotlin
    class Queue {  
      
      private val first = ArrayDeque<Int>()  
      private val second = ArrayDeque<Int>()  
      
      // Insert element v at the end of the queue.  
      fun enqueue(v: Int) {  
        while (first.isNotEmpty()) second.add(first.removeLast())  
        first.add(v)  
        while (second.isNotEmpty()) first.add(second.removeLast())  
      }  
      
      // Remove the element from the front of the queue.  
      fun dequeue(): Int? {  
        return if (first.isEmpty()) null else first.removeLast()  
      }  
      
      // Return the element at the front of the queue.  
      fun front(): Int? {  
        return if (first.isEmpty()) null else first.last()  
      }  
      
      // Number of elements in the queue.  
      fun size(): Int = first.size  
    }
    ```

??? "Second approach"

    In this approach, we use recursion as an implicit stack, allowing us to get away with using just one explicit stack.

    Let's start with handing the base case of $\text{enqueue}$ when the queue is empty. We simply $\text{push}$ the element to the underlying stack $s$.

    For all subsequent $\text{enqueue}$s, we $\text{pop}$ the stack, recursively call the $\text{enqueue}$ and then $\text{push}$ back the element we just popped.

    ```kotlin
    class Queue {  
      
      private val stack = ArrayDeque<Int>()  
      
      // Insert element v at the end of the queue.  
      fun enqueue(v: Int) {  
        if (stack.isEmpty()) {  
          stack.add(v)  
          return  
        }  
        val previous = stack.removeLast()  
        enqueue(v)  
        stack.add(previous)  
      }  
      
      // Remove the element from the front of the queue.  
      fun dequeue(): Int? {  
        return if (stack.isEmpty()) null else stack.removeLast()  
      }  
      
      // Return the element at the front of the queue.  
      fun front(): Int? {  
        return if (stack.isEmpty()) null else stack.last()  
      }  
      
      // Number of elements in the queue.  
      fun size(): Int = stack.size  
    }
    ```

## Unit tests

??? "Expand"

    ```kotlin
    import org.example.org.example.Queue  
    import org.junit.jupiter.api.Assertions.*  
    import org.junit.jupiter.api.BeforeEach  
    import org.junit.jupiter.api.Test  
      
    class QueueUsingStacksTest {  
      
      private lateinit var queue: Queue  
      
      @BeforeEach  
      fun setup() {  
        queue = Queue()  
      }  
      
      @Test  
      fun `test basic enqueue and dequeue`() {  
        queue.enqueue(1)  
        queue.enqueue(2)  
        queue.enqueue(3)  
      
        assertEquals(1, queue.dequeue(), "First enqueued should be first dequeued")  
        assertEquals(2, queue.dequeue())  
        assertEquals(3, queue.dequeue())  
      }  
      
      @Test  
      fun `test peek operation`() {  
        queue.enqueue(10)  
        queue.enqueue(20)  
      
        assertEquals(10, queue.front(), "Peek should return the front element without removing it")  
        assertEquals(10, queue.front(), "Subsequent peek should still return the same element")  
        assertEquals(10, queue.dequeue(), "Dequeue after peek should return the same element")  
      }  
      
      @Test  
      fun `test isEmpty on new queue`() {  
        assertTrue(queue.size() == 0, "New queue should be empty")  
      }  
      
      @Test  
      fun `test dequeue from empty queue`() {  
        assertNull(queue.dequeue(), "Dequeueing from an empty queue should return null")  
      }  
      
      @Test  
      fun `test interleaved enqueue and dequeue`() {  
        // Enqueue 1, 2  
        queue.enqueue(1)  
        queue.enqueue(2)  
        assertEquals(1, queue.dequeue())  
      
        // Enqueue 3  
        queue.enqueue(3)  
        assertEquals(2, queue.dequeue(), "Should maintain FIFO even when new elements are added mid-way")  
        assertEquals(3, queue.dequeue())  
        assertTrue(queue.size() == 0)  
      }  
      
      @Test  
      fun `test large number of operations`() {  
        val list = (1..1000).toList()  
        list.forEach { queue.enqueue(it) }  
      
        list.forEach {  
          assertEquals(it, queue.dequeue())  
        }  
        assertTrue(queue.size() == 0)  
      }  
    }
    ```
